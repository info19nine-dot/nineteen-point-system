# プロジェクト引き継ぎ・進捗報告書

## 1. プロジェクト概要
**名称**: ポイントカードアプリ (Point Card Web App)
**目的**: 店舗独自のポイント管理システムの構築。会員証のデジタル化と、スタッフによるポイント付与・決済の効率化を目指す。
**コンセプト**: 
- **「PayPayライク」な操作感**: 直感的でアニメーションを多用した、心地よいUI/UX。
- **プレミアムなデザイン**: ティールグリーン (`#2b9b96`) を基調とした、高級感と清潔感のある「会員証」デザイン。

## 2. ユーザー（クライアント）の要望と要件
本プロジェクトにおいて、ユーザーが特に重視しているポイントは以下の通りです。

### デザイン・UX要件
- **モバイルファースト**: スマホでの操作を最優先（ボタンサイズ、タップ領域、スワイプ操作）。
- **視認性と安心感**: 
  - 会員証には名前を大きく表示。
  - アラートなどの無機質な確認画面は避け、リッチなポップアップアニメーションを採用。
- **誤操作防止**: 
  - 支払いQRコード提示時、会員側で「金額確定」操作を行わないとQRコードが有効にならない（ぼかし処理）仕組みの導入。
  - 誤スキャンや意図しない決済を防ぐUXフロー。

### 管理・運用要件
- **柔軟なメニュー管理**: スタッフが管理画面から「コース（メニュー）」を追加・編集・削除できること。
- **拡張性のあるレイアウト**: コースボタンが増えてもレイアウトが崩れない「横スクロール（カルーセル）」の実装。
- **検索機能**: IDや名前で会員を即座に検索できる機能（デモ実装済み）。

## 3. システム技術構成
### Tech Stack
- **Frontend**: React (Vite)
- **Language**: TypeScript
- **Styling**: Tailwind CSS (独自ユーティリティ含む)
- **Routing**: React Router Dom
- **Icons**: Lucide React
- **State Management**: React Context (`MockContext`)

### 主要ディレクトリ構造
```
src/
├── components/      # 再利用可能なUIコンポーネント
├── contexts/        # MockContextなど状態管理
├── pages/
│   ├── admin/       # 管理者・スタッフ向け画面 (Dashboard, Settings, SearchResults)
│   ├── auth/        # 認証画面 (Login, Register)
│   └── member/      # 会員向け画面 (CardHome)
└── styles/          # Tailwind設定、グローバルCSS
```
