店舗・会員統合管理アプリ 開発企画書プロジェクト名: Loyalty Connect Project作成日: 202X年X月X日バージョン: 1.1 (Auth & Flow Update)1. 企画概要1.1 背景・目的店舗運営における「会員管理（CRM）」と「ポイント付与・利用」のフローをデジタル化し、業務効率化と顧客ロイヤリティの向上を図る。物理的な会員カードを廃止し、スマートフォン完結型のスムーズな体験を提供する。1.2 アプリケーション形態Webアプリ（PWA推奨）: アプリストアを経由せず、ブラウザ（Safari/Chrome）で即座に利用可能なWebアプリケーション。レスポンシブ対応: スマートフォン（iOS/Android）での利用を主とし、管理画面はタブレット/PCでも操作可能とする。1.3 ターゲットユーザー店舗管理者（Store Admin）: 受付・会計を行う現場スタッフ。会員（Member）: 来店客。自身のスマホでポイント管理を行う。2. システム構成図 (Architecture)モダンかつサーバーレスな構成を採用し、開発コストの圧縮とスケーラビリティを両立する。コード スニペットgraph TD

Frontend: React (Vite) + Tailwind CSSBackend / Database: SupabaseHosting / CI/CD: Vercel (GitHub連携による自動デプロイ)3. 機能要件詳細3.1 共通・認証機能（更新）店舗ログイン: 店舗スタッフ向けのメールアドレス/パスワード認証。会員ログイン (Social Login):対応プロバイダ: LINE, Google, X (Twitter), メールアドレス。アカウント紐付け: 各SNSのUIDと会員データを紐付け、機種変更や再ログイン時にデータを引き継げる仕様とする。セッション管理: Supabase Authを利用した永続的なログイン状態の維持。3.2 店舗管理アプリ (Store Panel)URLパス: /admin (要ログイン)機能カテゴリ詳細仕様備考ダッシュボード会員検索バー、コース選択パネル、直近の取引履歴を表示。会員検索IDまたは氏名によるインクリメンタルサーチ。DB上のprofilesテーブルを検索。コース管理付与するポイントコースの選択ボタン表示。【管理機能】 新規コースの追加、既存コースの編集（名称・pt数）、削除機能。初期値: 70分/100分QR発行選択したコースに基づき、会員に読み取らせるための「付与用QR」を動的に生成・表示。スキャン機能カメラを起動し、会員の「会員証QR」または「利用用QR」を読み取る。読み取り成功時、自動でDB更新処理を実行。jsQR等を使用ポイント調整任意のポイント数を入力し、手動で付与・減算する機能（イレギュラー対応用）。3.3 会員用アプリ (Member Wallet)URLパス: /card (要ログイン)機能カテゴリ詳細仕様備考会員証表示搭乗券（Boarding Pass）風デザイン。現在のランク、保有ポイントをリアルタイム表示。Supabase Realtimeポイント獲得「ポイントを貯める」ボタンでカメラ起動。店舗提示のQRを読み取りポイント加算。ポイント利用利用ポイント数を入力フォームに入力 → 決済用QRコードを生成・表示。履歴確認自身の来店履歴、ポイント増減履歴をリスト表示。4. データベース設計 (Supabase / PostgreSQL)① profiles (会員情報)id (uuid, PK): Supabase Auth (auth.users) IDと紐付けmember_code (text, unique): 検索用会員ID (例: 829013)display_name (text): 表示名email (text): 連絡用メールアドレスcurrent_points (int8): 現在の保有ポイントrank (text): Member, Gold, VIPなどprovider (text): 登録経路 (line, google, twitter, email)② courses (コースマスタ)店舗側で管理・追加・編集するマスタデータ。id (int8, PK)title (text): コース名 (例: "70分", "100分")points (int8): 付与ポイント数 (例: 1000, 1500)is_active (bool): 有効/無効フラグcreated_at (timestamp)③ transactions (取引履歴)ポイントの動きを全て記録するログ。id (int8, PK)user_id (uuid, FK): 対象会員type (text): 'EARN' (獲得) / 'USE' (利用)amount (int8): 変動ポイント数course_id (int8, FK, nullable): どのコースで獲得したかcreated_at (timestamp)5. UI/UX デザイン仕様デザインコンセプトメインカラー: ティールグリーン (#2b9b96)フォント: 視認性の高いゴシック体（Inter / Noto Sans JP）モチーフ:店舗側: 誤操作を防ぐ、大きめのボタンと明確なコントラスト（Slate Grayベース）。会員側: 旅・非日常感を感じさせる「航空券」モチーフ。チケットの切り欠き（パンチ穴）や点線をCSSで再現。画面遷移図・認証フロー（更新）1. ログイン画面 (/login)アプリ起動時の初期画面。SNSログインボタン: 「LINEでログイン」「Googleでログイン」「X (Twitter)でログイン」※既存アカウントがあればログイン、なければ新規作成フローへ内部的に分岐。メールログイン: メールアドレス・パスワード入力欄 ＋ 「ログイン」ボタン。新規登録への導線:「新規会員登録はこちら」ボタンを配置。※押下すると、表示を「新規登録モード」に切り替える、または新規登録画面へ遷移する。2. 新規登録画面 (/register)ログイン画面の「新規会員登録」ボタンから遷移。SNSで登録（初期ログイン）:「LINE / Google / X で始める」ボタン。※ワンタップで認証し、profiles テーブルに初期データを作成してログイン完了とする。メールで登録:メールアドレス・パスワード入力欄 ＋ 「新規登録」ボタン。3. 管理トップ (/admin/dashboard)[会員検索] / [コース選択] / [履歴]4. 設定画面 (/admin/settings)コース追加・編集フォーム5. 会員証画面 (/card)[カード表示][カメラ起動モーダル][利用ポイント入力] → [QR表示モーダル]6. 開発環境・ワークフローソースコード管理 (GitHub):リポジトリ: loyalty-app-frontendブランチ運用: main (本番), develop (開発)ホスティング・デプロイ (Vercel):GitHubの main ブランチにプッシュされた時点で、Vercelが自動ビルド・デプロイを実行。環境変数 (.env) にて Supabase の API URL / Key を管理。開発言語・ライブラリ:React: UI構築Tailwind CSS: スタイリングSupabase JS Client: DB接続・認証・リアルタイムリスナーReact Router: 画面遷移管理React QR Code / QR Reader: QR機能7. 実装ステップ（ロードマップ）環境構築:Vite (React) プロジェクト作成。Supabase プロジェクト作成・テーブル定義 (SQL実行)。認証機能実装 (Auth):Supabase Auth設定 (Google, Twitter, LINEプロバイダ有効化)。ログイン画面・新規登録画面のUI実装と紐付け処理。ベースUI実装:Tailwind CSS設定。V4デザイン（店舗・会員）のコンポーネント実装。機能実装（バックエンド連携）:コース一覧の取得・追加・編集機能の実装。会員データの表示・リアルタイム更新実装。QR・カメラ連携:QR生成ロジックと、カメラスキャン時のDB更新処理。デプロイ・テスト:Vercelへのデプロイと実機（スマホ）テスト。以上が本プロジェクトの正式な企画書となります。この仕様に基づき、開発・実装を進めていきます。
